# Laundry Day Battle

A 2D pixel-art, **local split-screen 2-player** competitive game built with **Phaser 4 + TypeScript + Vite**. Two players race to process laundry through a **Wash → Dry → Fold** pipeline while wind, rain, and runaway socks get in the way. Most points when the clock runs out wins.

## Quick start

```bash
npm install
npm run preprocess   # process source art in /Assets -> /public/assets (required once)
npm run dev          # open http://localhost:5173
```

Build for production:

```bash
npm run build        # tsc typecheck + vite build -> /dist
npm run preview
```

> **`npm run preprocess` is required before the first run.** The source art in
> `/Assets` is AI-generated on a solid **magenta chroma-key background** at very
> high resolution. The preprocessor (`scripts/preprocess.mjs`) keys out the
> magenta to transparency (with a soft edge feather), downscales each image with
> a **premultiplied-alpha box filter** (so transparent magenta never bleeds into
> sprite edges), strips spaces from filenames (`P1 - A1.png` → `P1-A1.png`), and
> writes the results to `public/assets/`. It also emits
> `src/generated/assetSizes.ts` so the asset manifest can derive every
> spritesheet's frame size automatically.

## Controls

| Action | Player 1 | Player 2 |
|---|---|---|
| Move | `W A S D` | Arrow keys |
| Grab / drop / interact | `Space` | `Enter` |
| Action (wash mash / fold timing / use power-up) | `F` | `.` or `Right Shift` |

Menu / Character Select advance with `Enter` / `Space` / click / tap.

### Tablet & mobile (touch)

The game is touch-compatible. On any touch device, **on-screen controls appear automatically** during play — each split-screen half gets a floating **virtual joystick** on its outer edge and **GRAB** / **ACT** buttons toward the center divider (so two people can share a tablet, one per side). It's fully multitouch.

- Play in **landscape** — a "rotate your device" prompt shows in portrait, since split-screen needs the width.
- Tapping **PLAY** requests fullscreen and tries to lock landscape orientation.
- The viewport is locked against pinch-zoom and scroll/overscroll, with iOS safe-area support.
- Append `?touch=1` to the URL to force the on-screen controls on a desktop browser (handy for testing).

## How to play

1. **Grab** a dirty item from the shared basket in the middle (both players compete for it).
2. **Wash** at your machine — mash the action key (or hold it) to fill the scrub meter. Item becomes **WET**.
3. **Hang** the wet item on your clothesline. It dries over time (watch the blue bar). Item becomes **DRY**.
4. **Fold** the dry item at your table — stop the moving cursor in the green zone (hit the yellow center for a **PERFECT** bonus). Item becomes **FOLDED** → points!
5. **Socks** need a pair: fold one sock and it waits by your pile; fold a second to complete the pair for a big bonus.

Watch out: **wind** knocks hung clothes to the ground, **rain** re-wets everything on the line, and a **runaway sock** event teleports your pending sock across the yard. In the final 30 seconds, **Rush Hour** speeds everything up. Grab **power-ups** (Soap Bomb, Mischief Wind, Sock Snatch, Speedy Spin) for an edge.

## Project layout

```
Assets/                  source art (committed; magenta chroma-key, hi-res)
public/assets/           processed art (generated, committed so Vercel deploys work)
scripts/
  preprocess.mjs         chroma-key + downscale pipeline
  e2e.mjs, e2e2.mjs      headless puppeteer smoke / pipeline tests
  shot.mjs               headless screenshot capture
src/
  main.ts                Phaser game config + scene list
  config/                gameConfig.ts (all tuning), assetManifest.ts
  generated/             assetSizes.ts (auto-generated by preprocess)
  types/                 enums, interfaces, event keys
  scenes/                Boot, Preload, Menu, CharacterSelect, Game, UI, GameOver
  entities/              Player, ClothesItem, PowerUp, stations/*
  systems/               InputManager, ScoreSystem, DisruptionSystem, Sfx
```

All gameplay numbers live in [`src/config/gameConfig.ts`](src/config/gameConfig.ts) (`CFG`). Set `BEST_OF` to `3` there for a best-of-3 match (defaults to `1` for quick iteration).

## Tests

Headless verification using the installed Chrome via `puppeteer-core` (the dev
server must be running on port 5173):

```bash
npm run dev            # in one terminal
node scripts/e2e.mjs   # drives a full grab→wash→hang→dry→fold→score pipeline
node scripts/e2e2.mjs  # verifies sock pairing, wind, rain, sock-event, power-ups
```

A dev-only hook (`window.LDB`, enabled when `import.meta.env.DEV`) exposes the
Phaser game so the tests can inspect/teleport. It is not present in production builds.

## Notes & decisions

- **Logo (`F1`)**: unlike the other art it sits on a solid **white** (not magenta) background, so it gets a flood-fill background removal (`scripts/lib/imagekit.mjs`, applied in `preprocess.mjs`). If you only have the already-processed `public/assets/F1.png` and not the source, run `node scripts/fix-logo.mjs` to strip it in place.
- **DRY vs WET sprites**: per the brief, DRY reuses the WET art (`B2`), distinguished by the full drying meter plus a sparkle effect — no extra asset needed.
- **Action sprites (`P*-A4`)**: these source frames bake the washing machine / folding table into the character. To avoid a doubled station on screen, the player keeps their idle/carry sprite during mini-games and feedback is shown via particles + the on-station meter.
- **Sound**: lightweight synthesised WebAudio blips (`src/systems/Sfx.ts`) — no audio assets required.
- **Best score** is persisted to `localStorage`.
